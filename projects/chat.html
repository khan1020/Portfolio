Building Chat Without WebSockets

"Use WebSockets for real-time!" they said. But my shared hosting doesn't support WebSockets. So I figured out how to build a responsive chat experience using good old HTTP polling.

Turns out, with smart polling intervals and efficient queries, you can build a chat app that feels instant â€” even on basic PHP hosting. The secret? Poll every 2 seconds, but only fetch messages newer than the last one you received.

How It Works

Username System

Pick a username to join. Each user gets a randomly assigned color for easy identification.

Instant Messaging

Messages appear within 2 seconds. No page refresh needed. AJAX handles everything.

Message History

See the last 50 messages when you join. New messages scroll automatically.

User Colors

Each username gets a consistent color. Makes it easy to follow conversations.

Why AJAX Polling?
WebSockets need persistent connections and special server configuration. AJAX polling works everywhere: shared hosting, XAMPP, anywhere PHP runs. For a demo or small-scale app, it's perfect.

The Challenge: Race Conditions

What happens when two people send a message at the exact same time? The database handles it with auto-increment IDs. Each message gets a unique ID, and we fetch based on that â€” no duplicates, no missing messages.
