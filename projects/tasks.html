Why Another To-Do App?

I know, I know. "The world doesn't need another to-do app." But here's the thing — building a task manager is the perfect way to learn CRUD operations properly. Every feature teaches you something: filtering teaches SQL WHERE clauses, priorities teach data validation, categories teach foreign key relationships.

Plus, I actually use this one. It's on my localhost, and I track my learning goals with it.

Database Design

I spent a good hour just on the database schema. Getting the relationships right (tasks → categories) saved me a ton of headaches later:

CREATE TABLE categories ( id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(50) NOT NULL, color VARCHAR(7) DEFAULT '#3b82f6' ); CREATE TABLE tasks ( id INT AUTO_INCREMENT PRIMARY KEY, title VARCHAR(200) NOT NULL, description TEXT, category_id INT, priority ENUM('low', 'medium', 'high') DEFAULT 'medium', due_date DATE, status ENUM('pending', 'completed') DEFAULT 'pending', created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (category_id) REFERENCES categories(id) );

The Filter Query That Took Forever

Building the multi-filter functionality was tricky. Users can filter by status AND category AND priority simultaneously. Here's how I handled it:

// Build dynamic WHERE clause $where = []; $params = []; if ($status) { $where[] = "status = ?"; $params[] = $status; } if ($category_id) { $where[] = "category_id = ?"; $params[] = $category_id; } if ($priority) { $where[] = "priority = ?"; $params[] = $priority; } $sql = "SELECT * FROM tasks"; if (count($where) > 0) { $sql .= " WHERE " . implode(" AND ", $where); }

Lessons Learned

✅ ENUM is underrated

For fields with fixed options (status, priority), ENUM in MySQL is perfect. It enforces valid values at the database level.

✅ Foreign keys save lives

When I added ON DELETE CASCADE, accidentally deleting a category also deletes its tasks. Without it? Orphaned data everywhere.

✅ JavaScript for interactions, PHP for logic

The checkbox to mark tasks complete uses AJAX. The page doesn't reload. Small UX wins make apps feel polished.
