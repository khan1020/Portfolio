Why OTP?

Passwords get stolen. Phishing works. OTP adds a second layer ‚Äî something you have (your phone) in addition to something you know (password).

I built this to understand the OTP flow: generate code, set expiration, validate input, handle wrong attempts, allow resend with rate limiting.

The Flow

Enter Phone

User enters their phone number to receive OTP.

Send OTP

6-digit code generated and "sent" (shown in demo).

Enter Code

User types the 6 digits. Auto-advances between boxes.

Verify

System validates code. Grant access or show error.

Generating Secure OTPs

OTPs need to be random but also short enough to type easily:

function generateOTP($length = 6) { // Use cryptographically secure random return str_pad( random_int(0, pow(10, $length) - 1), $length, '0', STR_PAD_LEFT ); } // Store with expiration $_SESSION['otp'] = [ 'code' => generateOTP(), 'expires' => time() + (5 * 60), // 5 minutes 'attempts' => 0 ];

The OTP Input UX

6 separate boxes that auto-advance as you type:

// Auto-advance to next input inputs.forEach((input, index) => { input.addEventListener('input', (e) => { if (e.target.value && index < inputs.length - 1) { inputs[index + 1].focus(); } }); // Handle backspace input.addEventListener('keydown', (e) => { if (e.key === 'Backspace' && !e.target.value && index > 0) { inputs[index - 1].focus(); } }); });

What I Learned

üîê OTPs must expire

5 minutes is standard. After that, require a new code. Never let OTPs be reused.

üîê Limit verification attempts

3 wrong attempts = lock out and require new OTP. Prevents brute forcing a 6-digit code.

üîê Rate limit resends

Don't let users spam "resend". 60-second cooldown prevents SMS bombing (and saves money).
