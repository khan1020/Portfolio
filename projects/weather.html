The Story

I remember checking the weather on my phone and thinking, "I could build this." Famous last words, right? But seriously ‚Äî weather apps are deceptively simple-looking while teaching you fundamental skills: API calls, async JavaScript, dynamic UI updates.

This project taught me more about fetch(), Promises, and error handling than any tutorial ever did. Because when your API call fails at 2 AM, you learn fast.

API Note: This demo uses OpenWeatherMap's free tier. The API key in the code is for demo purposes. For production, you'd want to proxy through your own backend.

Features

City Search

Type any city name to get current weather. Handles typos gracefully with error messages.

Geolocation

Click "Use My Location" to get weather for your current position using the browser's Geolocation API.

5-Day Forecast

See what's coming with a scrollable 5-day forecast showing high/low temps and conditions.

Dynamic Background

Background color changes based on weather conditions ‚Äî sunny, cloudy, rainy, or snowy.

The Tricky Part: Async/Await

Before this project, I was still using .then().catch() chains everywhere. This project pushed me to finally learn async/await properly:

async function getWeather(city) { try { // Show loading state showLoading(); const response = await fetch( `${API_URL}?q=${city}&appid=${API_KEY}&units=metric` ); if (!response.ok) { throw new Error('City not found'); } const data = await response.json(); displayWeather(data); } catch (error) { showError(error.message); } finally { hideLoading(); } }
